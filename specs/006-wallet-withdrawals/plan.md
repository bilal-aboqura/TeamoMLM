## Summary

Build the user-facing Wallet & Withdrawals page at `/dashboard/wallet`. The module adds: a `withdrawal_requests` table with full audit trail support, a `user_submit_withdrawal` PostgreSQL DEFINER function for atomic balance-decrement + request-insert, two admin RPC stubs (`admin_approve_withdrawal`, `admin_reject_withdrawal`) for future admin panel integration, a Zod schema for server-side validation, a Server Action wrapping the RPC, and three UI components (stats cards, withdrawal form, history table) following the RTL-first design constitution.

## Phase 1: Implementation Design

### Layer 1 — Database Migration (`20260406000010`)

**File**: `supabase/migrations/20260406000010_create_withdrawal_requests.sql`

**Operations** (in order):
1. Extend `financial_audit_log.record_type` CHECK constraint to include `'withdrawal_request'`
2. Create `public.withdrawal_requests` table (see data-model.md for full schema)
3. Create `updated_at` auto-trigger on `withdrawal_requests`
4. Create indexes: `(user_id, created_at DESC)` and partial `WHERE status = 'pending'`
5. Enable RLS on `withdrawal_requests`
6. Create RLS policy: `withdrawal_requests_select_own` — `FOR SELECT USING (auth.uid() = user_id)`
7. Create `public.user_submit_withdrawal(p_user_id, p_amount, p_payment_details)` — DEFINER function
   - `FOR UPDATE` lock on `users` row
   - Check `status = 'active'` → RAISE `'suspended'`
   - Check `wallet_balance >= p_amount` → RAISE `'insufficient_balance'`
   - Decrement `wallet_balance`
   - Insert `withdrawal_requests` row
   - Returns `UUID` (new request id)
8. Create `public.admin_approve_withdrawal(p_request_id, p_admin_id)` — DEFINER function
   - `FOR UPDATE` lock, pending-guard, status update, audit log insert
9. Create `public.admin_reject_withdrawal(p_request_id, p_admin_id, p_reason)` — DEFINER function
   - `FOR UPDATE` lock, pending-guard, status update, balance refund, audit log insert

### Layer 2 — Zod Validation Schema

**File**: `lib/validations/wallet-schemas.ts`

```typescript
export const submitWithdrawalSchema = z.object({
  amount: z
    .string()
    .transform((val) => parseFloat(val))
    .pipe(
      z.number()
        .min(1.00, "الحد الأدنى للسحب هو $1.00")
        .max(999999.99, "المبلغ تجاوز الحد المسموح")
        .refine(
          (n) => Number.isFinite(n) && Math.round(n * 100) === n * 100,
          "المبلغ يجب أن يكون بحد أقصى خانتين عشريتين"
        )
    ),
  payment_details: z
    .string()
    .trim()
    .min(1, "تفاصيل وسيلة الدفع مطلوبة")
    .max(200, "الحد الأقصى 200 حرف"),
});
```

### Layer 3 — Server Action

**File**: `app/dashboard/wallet/actions.ts`

```
"use server"
export type WithdrawalActionResult =
  | { success: false; idle: true }
  | { success: true }
  | { error: { field: string; message: string } }

export async function submitWithdrawal(
  _prevState: WithdrawalActionResult,
  formData: FormData
): Promise<WithdrawalActionResult>
```

**Logic sequence**:
1. Extract `amount` (string from form) and `payment_details` from formData
2. Zod validate → return field error if invalid
3. `createClient().auth.getUser()` → 401 error if no session
4. `createAdminClient().rpc("user_submit_withdrawal", { p_user_id, p_amount, p_payment_details })`
5. Map RPC error signals:
   - `'suspended'` → Arabic "حسابك موقوف. لا يمكنك تقديم طلبات سحب."
   - `'insufficient_balance'` → Arabic "رصيدك غير كافٍ لإتمام هذا الطلب"
   - Other → generic Arabic "حدث خطأ، يرجى المحاولة مرة أخرى"
6. Return `{ success: true }` on clean RPC response

### Layer 4 — UI Components

#### `WalletStatsCards.tsx` (RSC)

Props: `{ availableBalance: number, totalWithdrawn: number, pendingWithdrawals: number }`

Data source (fetched in `page.tsx`):
- `availableBalance` ← `users.wallet_balance` for current user
- `totalWithdrawn` ← `SUM(amount) FROM withdrawal_requests WHERE user_id = X AND status = 'approved'`
- `pendingWithdrawals` ← `SUM(amount) FROM withdrawal_requests WHERE user_id = X AND status = 'pending'`

Render: 3 cards in `grid grid-cols-1 sm:grid-cols-3 gap-4`. Each card:
- `bg-white rounded-2xl p-6 border border-slate-100 shadow-[0_2px_10px_rgba(0,0,0,0.02)]`
- Title in `text-sm text-slate-500`
- Amount in `<span dir="ltr">` with emerald highlight for Available Balance

#### `WithdrawalForm.tsx` (`"use client"`)

Justification: `useActionState`, `useFormStatus`, `useRouter`, form ref reset, toast state.

State:
- `[state, formAction] = useActionState(submitWithdrawal, initialState)`
- `formRef = useRef<HTMLFormElement>`
- `toast: string | null` state for success message (auto-dismisses via `setTimeout`)

`useEffect` on `state`:
```typescript
if ("success" in state && state.success) {
  formRef.current?.reset();            // FR-011(a): clear fields
  setToast("تم إرسال طلب السحب بنجاح"); // FR-011(b): success toast
  setTimeout(() => setToast(null), 3000);
  router.refresh();                    // FR-011(d): refresh cards + history
}
```

Suspended state: page passes `isSuspended: boolean` prop; if true, form is replaced with a banner:
```html
<div class="bg-amber-50 rounded-2xl p-6 ...">حسابك موقوف...</div>
```

Zero balance state: submit button `disabled` when `availableBalance === 0` (FR-012).

Client-side validation (FR-007, FR-013):
- Amount: > 0, ≤ availableBalance, ≤ 2 decimal places
- On blur, show inline error below the input

#### `WithdrawalsTable.tsx` (RSC)

Props: `{ requests: WithdrawalRow[] }` — data fetched in parent `page.tsx`.

Columns: Date (Arabic locale), Amount (`dir="ltr"`), Payment Details (truncated via `truncate max-w-[12rem]`), Status badge.

Status badges:
- `pending`: `bg-amber-50 text-amber-700 rounded-full px-2 py-0.5`
- `approved`: `bg-emerald-50 text-emerald-700 rounded-full px-2 py-0.5`
- `rejected`: `bg-red-50 text-red-700 rounded-full px-2 py-0.5`

If `rejection_reason` present on rejected row: show below the badge as `text-xs text-slate-500`.

Empty state: full-width `bg-white rounded-2xl p-12 text-center` with "لا توجد طلبات سحب حتى الآن".

#### `page.tsx` (RSC)

Fetches all three data points in parallel via `Promise.all`:
```typescript
const [userRow, withdrawalAgg, requests] = await Promise.all([
  supabase.from("users").select("wallet_balance, status").eq("id", user.id).single(),
  supabase.from("withdrawal_requests")
    .select("amount, status")
    .eq("user_id", user.id),
  supabase.from("withdrawal_requests")
    .select("id, amount, payment_details, status, rejection_reason, created_at")
    .eq("user_id", user.id)
    .order("created_at", { ascending: false }),
]);
```

Computes `totalWithdrawn` and `pendingWithdrawals` client-side from `withdrawalAgg` data.

Page layout:
```
<main dir="rtl" class="p-6 space-y-8 max-w-3xl mx-auto">
  <WalletStatsCards ... />
  <WithdrawalForm availableBalance={...} isSuspended={...} />
  <WithdrawalsTable requests={...} />
</main>
```
